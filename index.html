The % Operator in C++

Complete guide — from a single main() you can paste into any compiler, all the way to C++20 modules.

Part 1
Simple examples — copy, paste, compile
Every example below is one file. Works in any online compiler: onlinegdb.com, godbolt.org, replit.com, https://www.codechef.com/ide?authuser=0

What does % actually do?

% gives you the remainder after integer division. Think of it like splitting items into equal groups — whatever is left over is the result.

17 ÷ 5 = 3 groups of 5 (= 15), with 2 left over → 17 % 5 = 2

example1.cpp — what % does 
#include <iostream>
using namespace std;

int main() {
    // % = remainder after division
    cout << "17 % 5  = " << 17 % 5  << endl;  // 2  (5 goes into 17 three times = 15, left over = 2)
    cout << "10 % 3  = " << 10 % 3  << endl;  // 1  (3 goes into 10 three times = 9,  left over = 1)
    cout << "9  % 3  = " << 9  % 3  << endl;  // 0  (divides exactly — nothing left over)
    cout << "7  % 2  = " << 7  % 2  << endl;  // 1
    cout << "100 % 7 = " << 100 % 7 << endl;  // 2  (7 × 14 = 98, left over = 2)

    return 0;
}
output
17 % 5 = 2 10 % 3 = 1 9 % 3 = 0 7 % 2 = 1 100 % 7 = 2
Rule: if the result is 0, the number divides evenly. If it's anything else, there's a remainder.

Even and odd check

The most common use of %. Every even number divides by 2 with no remainder. Every odd number leaves 1.

example2_cout.cpp — using cout 
#include <iostream>
using namespace std;

int main() {
    int x = 7;

    // n % 2 == 0 means nothing is left over → even
    // n % 2 != 0 means 1 is left over       → odd
    if (x % 2 == 0) {
        cout << x << " is even" << endl;
    } else {
        cout << x << " is odd" << endl;
    }

    return 0;
}
output
7 is odd
cout vs printf — what's the difference?
Both print to the screen. cout is C++ style — you chain things with <<. printf is C style — you use a format string with placeholders like %d for integers, %s for strings. Both work fine. printf is often shorter for formatted output. cout is safer with types. In a real codebase pick one and stick to it.
example2_printf.cpp — same thing using printf 
#include <cstdio>    // needed for printf

int main() {
    int x = 7;

    if (x % 2 == 0) {
        printf("%d is even\n", x);  // %d is the placeholder for an integer
    } else {
        printf("%d is odd\n", x);
    }

    return 0;
}
output
7 is odd
Feature cout (C++ style) printf (C style)
Header needed <iostream> <cstdio> or <stdio.h>
Print an int cout << x; printf("%d", x);
New line endl or "\n" "\n" inside the string
Type safety Checked at compile time Not checked — wrong %d can crash
Readability Verbose with many variables Compact with many variables
Speed Slightly slower by default Slightly faster

Loop through numbers — even and odd

Using % inside a loop to label every number. A very common classroom pattern.

example3.cpp — loop with % check (cout + printf mixed) 
#include <iostream>
#include <cstdio>
using namespace std;

int main() {

    // cout version
    cout << "--- using cout ---" << endl;
    for (int i = 1; i <= 10; i++) {
        if (i % 2 == 0)
            cout << i << " is even" << endl;
        else
            cout << i << " is odd"  << endl;
    }

    // printf version — same logic, shorter lines
    printf("\n--- using printf ---\n");
    for (int i = 1; i <= 10; i++) {
        printf("%d is %s\n", i, (i % 2 == 0) ? "even" : "odd");
        //                    ^^ %s = string placeholder
    }

    return 0;
}
output
--- using cout --- 1 is odd 2 is even 3 is odd 4 is even 5 is odd 6 is even 7 is odd 8 is even 9 is odd 10 is even --- using printf --- 1 is odd 2 is even ... (same result)

FizzBuzz

The classic problem. The trick: check % 15 first — because if a number is divisible by both 3 and 5, the individual checks would catch it first and give the wrong answer.

example4.cpp — FizzBuzz 
#include <cstdio>

int main() {
    for (int i = 1; i <= 30; i++) {

        // MUST check 15 first — 15 is divisible by both 3 and 5
        if      (i % 15 == 0) printf("FizzBuzz\n");
        else if (i % 3  == 0) printf("Fizz\n");
        else if (i % 5  == 0) printf("Buzz\n");
        else                   printf("%d\n", i);
    }
    return 0;
}
output (first 16 lines)
1 2 Fizz 4 Buzz Fizz 7 8 Fizz Buzz 11 Fizz 13 14 FizzBuzz ← 15: divisible by both 3 and 5 16

Last digit of a number

n % 10 always gives the last digit. Combine with / to peel digits one at a time.

example5.cpp — digit extraction 
#include <cstdio>

int main() {
    int n = 4827;

    printf("Number: %d\n", n);
    printf("Last digit:   %d\n",  n % 10);          // 7
    printf("Second digit: %d\n",  (n / 10) % 10);   // 2  (shift right, then grab last)
    printf("Third digit:  %d\n",  (n / 100) % 10);  // 8
    printf("Fourth digit: %d\n",  (n / 1000) % 10); // 4

    // Sum all digits using a loop
    int sum = 0, temp = n;
    while (temp > 0) {
        sum  += temp % 10;   // grab last digit and add to sum
        temp /= 10;          // chop off last digit
    }
    printf("Digit sum: %d\n", sum);  // 4+8+2+7 = 21

    return 0;
}
output
Number: 4827 Last digit: 7 Second digit: 2 Third digit: 8 Fourth digit: 4 Digit sum: 21

Wrap-around (circular index)

When you reach the end of an array and want to go back to the start, % does the wrap automatically. Very useful in menus, ring buffers, and game loops.

example6.cpp — circular wrap 
#include <iostream>
using namespace std;

int main() {
    int size = 5;
    int index = 0;

    // Simulate stepping through a menu that loops back to start
    for (int step = 0; step < 12; step++) {
        cout << "Step " << step << ": index = " << index << endl;
        index = (index + 1) % size;  // 4 → 5 would go out of bounds, % wraps to 0
    }

    return 0;
}
output
Step 0: index = 0 Step 1: index = 1 Step 2: index = 2 Step 3: index = 3 Step 4: index = 4 Step 5: index = 0 ← wrapped back! Step 6: index = 1 Step 7: index = 2 ...
Part 2
Advanced — C++20 Modules
You do NOT need this to use %. This is for larger projects. Requires C++20 and specific compiler flags explained below.

Why do modules exist?

Before C++20, every file used #include to share code. That means the compiler literally copy-pastes the header file into every single source file that includes it. In a large project with hundreds of files, the same headers get re-parsed thousands of times — making builds very slow. Macros defined in one header can also accidentally affect other files.

C++20 modules fix this. You compile a module once into a Binary Module Interface file (.bmi / .pcm). Every file that imports it reads that pre-compiled binary — not the source — so it's much faster and macros can't leak.

When to use modules: Large projects (many files, shared libraries). For a classroom exercise or a single-file program — just use #include. Simpler is better.

The same even/odd example — module version

This requires two files. The module interface file and the main file.

File 1 of 2 — the module

checker.cppm 
// This declares this file as a module named "checker"
export module checker;

// "export" = this function is visible to anyone who imports this module
export bool isEven(int n) {
    return n % 2 == 0;
}

export bool isOdd(int n) {
    return n % 2 != 0;
}

// No "export" = private to this module. main.cpp cannot see this.
static int callCount = 0;

File 2 of 2 — the main program

main.cpp 
// "import" instead of #include — reads the pre-compiled BMI, not source
import checker;
#include <iostream>
using namespace std;

int main() {
    cout << isEven(4) << endl;  // 1 (true)
    cout << isOdd(7)  << endl;  // 1 (true)
    // callCount++;  ← compile error: not exported, not visible here
    return 0;
}

Build commands and what the flags mean

You cannot just run g++ main.cpp for modules. The module must be compiled first, and the compiler needs to know where to find it. Here is every flag explained.

build — Clang (recommended for modules) 
# Step 1 — compile the module interface into a BMI (.pcm file)
clang++ -std=c++20 --precompile checker.cppm -o checker.pcm

# Step 2 — compile the module interface into an object file (.o)
clang++ -std=c++20 -c checker.cppm -o checker.o

# Step 3 — compile main.cpp, tell it where the BMI is
clang++ -std=c++20 -fmodule-file=checker=checker.pcm -c main.cpp -o main.o

# Step 4 — link everything into the final executable
clang++ checker.o main.o -o program
build — GCC 14+ 
# GCC handles BMI discovery automatically with -fmodules-ts
g++ -std=c++20 -fmodules-ts checker.cppm -c -o checker.o
g++ -std=c++20 -fmodules-ts main.cpp     -c -o main.o
g++ checker.o main.o -o program

What each flag does — plain English

Flag What it does Why you need it
-std=c++20 Tells the compiler to use the C++20 standard Modules are a C++20 feature. Without this flag the compiler doesn't know what export module means and will error out.
--precompile Compiles a module interface into a Binary Module Interface (.pcm) The .pcm file is what consumers read instead of re-parsing your source. Without it, import checker has nothing to read.
-c Compile only — do not link yet You need separate .o files for each translation unit before linking them together at the end.
-fmodule-file=
name=path		 Tells Clang where the .pcm file for a named module lives When main.cpp says import checker, Clang needs to know where checker.pcm is on disk. This maps the name to the path.
-fmodules-ts Enables the modules Technical Specification in GCC GCC's module support is still being finalized. This flag opts into it. GCC auto-finds BMI files in a gcm.cache/ folder it creates.
-o filename Sets the output filename Without it the compiler uses a default name like a.out. Naming your outputs keeps multi-file builds organized.
Online compilers and modules: Most online compilers (onlinegdb, replit) do not support C++20 modules yet. To test module code you need Clang 16+ or GCC 14+ installed locally, or use Compiler Explorer at godbolt.org which has both.
